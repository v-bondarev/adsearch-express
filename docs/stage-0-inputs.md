# Данные и доступы для Этапа 0

Этот документ описывает, что нужно подготовить для исследования Active Directory и express.ms до начала разработки основной логики.

Секреты, пароли, токены и ключи не нужно передавать в чате или коммитить в репозиторий. Их нужно передавать через согласованный безопасный канал или размещать в `.env` на тестовом стенде.

## От администратора Active Directory

Нужно подготовить:

- LDAPS host и порт.
- Base DN для поиска.
- Список OU, которые включаются в поиск.
- Список OU, которые исключаются из поиска.
- Формат bind-пользователя: DN, UPN или `DOMAIN\user`.
- Тестовую учетную запись для bind с правами только на чтение нужных атрибутов.
- Способ проверки TLS-сертификата LDAPS: доверенный корпоративный CA или отдельный CA-файл.
- Правила определения отключенных пользователей.
- Правила определения сервисных, технических и скрытых учетных записей.
- Соответствие AD-атрибутов полям карточки, если оно известно заранее.
- Атрибут, в котором хранится фото сотрудника: например, `thumbnailPhoto`, `jpegPhoto` или другой.
- 2-3 тестовых ФИО сотрудников и контактов для проверки выдачи.
- Разрешение на получение полного списка атрибутов для тестовых объектов.

## От администратора express.ms

Нужно подготовить:

- Версию BotX API.
- Ссылку на актуальную документацию именно для вашей установки express.ms.
- BOT_ID.
- Способ получения BOT_SECRET_KEY или размещения секрета на стенде.
- Webhook URL, который нужно зарегистрировать.
- Сведения о доступности webhook: внутренний адрес или внешний HTTPS.
- HUID пользователей, которым разрешена команда `/clear_cache`.
- Ограничения BotX API на отправку изображений.
- Пример тестового сообщения боту для проверки webhook.

## Проверки Этапа 0

До разработки основной логики нужно подтвердить реальными запросами:

- структуру webhook от express.ms;
- формат авторизации исходящих запросов Bot -> BotX;
- успешный LDAPS bind;
- реальные LDAP-фильтры поиска;
- набор доступных AD-атрибутов для user и contact;
- поведение для точного совпадения, частичного совпадения, нескольких результатов и отсутствия результатов;
- получение и отправку фото сотрудника;
- ограничения по размеру и формату изображений в BotX API.
