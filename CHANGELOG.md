# Changelog

Все существенные изменения проекта документируются в этом файле.

## [Unreleased]

### Исправлено

- Healthcheck и `scripts/deploy.sh` теперь явно обходят HTTP proxy при проверке
  `http://127.0.0.1:8000/health`; в контейнерах также выставлены
  `NO_PROXY/no_proxy` для localhost.
- Обработка команд перенесена в фон: endpoint `/command` сразу подтверждает
  получение запроса, поэтому eXpress больше не показывает ошибку ожидания,
  пока выполняются LDAP-поиск и отправка карточек.
- LDAP-поиск вынесен из event loop, а запросы профилей eXpress выполняются
  параллельно. Заголовок результатов отправляется до загрузки ссылок профилей.
- Карточки сотрудников теперь выводятся без порядковых номеров; ФИО и названия
  заполненных полей выделяются жирным.
- Для записей с `thumbnailPhoto` фотография отправляется вместе с карточкой,
  а подпись отделяется от неё пустой строкой.
- Удалены неподдерживаемые параметры `pool_name`, `pool_size` и
  `pool_lifetime` из конструктора `ldap3.Server`, из-за которых любой поиск
  завершался ошибкой HTTP 500.
- Добавлен endpoint `GET /status` для служебной проверки бота со стороны
  eXpress.

### Добавлено

- Добавлен внутренний API `POST /api/search` с Bearer-авторизацией через
  `INTERNAL_API_TOKEN`, валидацией запроса, ограничением `SEARCH_LIMIT` и
  безопасной DTO-моделью без фотографий и LDAP DN.
- Добавлен отдельный контейнер `api`, доступный только через
  `127.0.0.1:8183`. Существующий контейнер `bot` и его порт не изменены.
- API подключён к внешней Docker-сети `adsearch-internal` с alias
  `adsearch-api`, чтобы другие контейнеры могли обращаться к нему по адресу
  `http://adsearch-api:8000`.
- `scripts/deploy.sh` автоматически создаёт сеть `adsearch-internal`, если она
  отсутствует.
- Добавлен документ `docs/kteam-express-adsearch-integration.md` с командами
  создания сети, настройками `.env`, контрактом API и рекомендациями для
  реализации клиента в `kteam-express`.
- API переиспользует LDAP-поиск через `asyncio.to_thread`, существующую
  блокировку запрещённых запросов и безопасное обогащение ссылками eXpress.
- В production API отказывается запускаться без `INTERNAL_API_TOKEN`.
- Добавлены тесты авторизации, валидации, ограничений выдачи, ошибок LDAP,
  структуры JSON и изоляции маршрутов BotX/API.

- В карточку сотрудника после офиса добавлен день рождения из
  `extensionAttribute5`. Год не выводится, пустое или некорректное значение
  скрывается. Перед названием поля отображается `📅`, а в день рождения перед
  датой добавляется `🎂`.
- Оптимизирован `scripts/deploy.sh`: используется fast-forward обновление,
  Docker layer cache и единая команда сборки/обновления контейнера. Фиксированная
  задержка удалена — скрипт опрашивает `/health` внутри контейнера каждую
  секунду, завершается сразу после готовности приложения и выводит последние
  логи при ошибке.
- Dockerfile перестроен для стабильного кеширования зависимостей: `APP_UID` и
  `APP_GID` больше не инвалидируют слой `pip install`, а код копируется после
  создания пользователя через `COPY --chown`. При неизменном
  `requirements.txt` Python-пакеты повторно не устанавливаются.

- **Тесты** — добавлен полный набор unit-тестов (pytest):
  - `tests/test_config.py` — тесты конфигурации
  - `tests/test_ldap_client.py` — тесты LDAP клиента и фильтров
  - `tests/test_cache.py` — тесты SQLite кэширования
  - `tests/test_formatter.py` — тесты форматирования карточек
  - `tests/test_botx_client.py` — тесты BotX HTTP клиента
  - `tests/test_main.py` — тесты эндпоинтов и обработки команд
  - 155 тестов (154 passed, 1 skipped)

- **Повторное использование конфигурации LDAP** — объект `ldap3.Server`
  создаётся один раз, а соединения гарантированно закрываются после запроса.

- **Graceful Shutdown** — при остановке приложения корректно освобождаются:
  - состояние LDAP-клиента (`ldap_client.close_pool()`)
  - HTTP клиент (`await close_http_client()`)

- **HTTP Client Reuse** — `httpx.AsyncClient` создаётся один раз и переиспользуется для всех запросов

- **Обновлён requirements.txt** — добавлены dev-зависимости: `pytest`, `pytest-asyncio`, `pytest-cov`
